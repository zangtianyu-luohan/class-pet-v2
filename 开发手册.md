## 1. 项目概述

### 1.1 背景
"班级OK萌宠"是一个面向教师的班级积分管理系统。学生通过课堂表现获得积分，积分驱动虚拟宠物成长升级，增加课堂趣味性。

### 1.2 功能清单

| 模块 | 功能 |
|------|------|
| 用户认证 | 注册、登录、修改密码、JWT 鉴权 |
| 班级管理 | 创建/编辑/删除班级、班级切换 |
| 学生管理 | 添加/编辑/删除学生、批量导入、搜索排序 |
| 积分系统 | 加分/减分、批量操作、积分记录、等级计算 |
| 排行榜 | 积分榜、等级榜、本周榜 |
| 勋章系统 | 创建勋章、颁发勋章、颁发记录 |
| 课堂工具 | 随机点名、计时器、随机分组 |
| 管理后台 | 数据库可视化管理面板 |

## 4. 后端开发详解

### 4.1 核心设计模式

**依赖注入（Dependency Injection）**：FastAPI 的核心设计，通过 `Depends()` 注入数据库会话和当前用户。

```python
# 获取当前用户的依赖
async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(security),
    db: AsyncSession = Depends(get_db),
) -> User:
    # 验证 JWT → 查询用户 → 返回
```

**异步全链路**：从路由到数据库全链路异步（`async/await`），高并发下不阻塞。

```python
@router.get("/", response_model=list[StudentOut])
async def list_students(
    class_id: int = Query(...),
    user: User = Depends(get_current_user),     # 依赖注入：鉴权
    db: AsyncSession = Depends(get_db),          # 依赖注入：数据库会话
):
    result = await db.execute(select(Student).where(...))
    return result.scalars().all()
```

### 4.2 认证流程

```
注册/登录 → 生成 JWT Token（有效期 7 天）
    ↓
前端存储 Token → localStorage
    ↓
每次请求 → Axios 拦截器自动添加 Authorization: Bearer <token>
    ↓
后端 → Depends(get_current_user) → 解码 JWT → 查询用户
```

### 4.3 等级计算

```python
LEVEL_EXP = [0, 100, 300, 600, 1000, 1500, 2100, 2800, 3600, 4500, 5500]

def calc_level(points):
    level = 1
    for i, exp in enumerate(LEVEL_EXP):
        if points >= exp:
            level = i + 1
    return level, points
```

### 4.4 管理后台 (sqladmin)

访问 `http://localhost:8000/admin` 可以在浏览器中直接管理所有数据库表：
- 查看、添加、编辑、删除任意记录
- 支持搜索和过滤
- 类似 Django Admin 的体验

## 5. 前端开发详解

### 5.1 状态管理（Pinia）

两个核心 Store：

```javascript
// auth.js — 认证状态
const token = ref(localStorage.getItem('token'))
const user = ref(JSON.parse(localStorage.getItem('user')))
// login() → 存储 token + user 到 localStorage
// logout() → 清除

// class.js — 班级状态
const classes = ref([])           // 所有班级
const currentClassId = ref(...)   // 当前选中班级
// fetchClasses() → 从 API 获取
// setCurrentClass(id) → 切换并持久化到 localStorage
```

### 5.2 路由守卫

```javascript
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  if (to.meta.requiresAuth && !authStore.isLoggedIn) next('/login')
  else if (to.meta.guest && authStore.isLoggedIn) next('/dashboard')
  else next()
})
```

### 5.3 Axios 拦截器

```javascript
// 请求拦截：自动加 token
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 响应拦截：统一错误处理
api.interceptors.response.use(
  res => res,
  err => {
    if (err.response?.status === 401) {
      localStorage.clear()
      window.location.href = '/login'
    }
    ElMessage.error(err.response?.data?.detail || '请求失败')
  }
)
```

### 5.4 Vite 代理（开发环境）

开发时前端 5173 端口、后端 8000 端口分开运行。通过 Vite 代理解决跨域：

```javascript
// vite.config.js
proxy: {
  '/api': {
    target: 'http://127.0.0.1:8000',
    changeOrigin: true,
  }
}
```

前端请求 `/api/auth/login` → Vite 代理到 `http://127.0.0.1:8000/api/auth/login`

## 9. 部署上线指南

### 9.1 注册账号

| 平台 | 地址 | 用途 |
|------|------|------|
| GitHub | github.com | 代码托管 |
| Supabase | supabase.com | 免费 PostgreSQL |
| Render | render.com | 后端托管 |
| Vercel | vercel.com | 前端托管 |

### 9.2 数据库 (Supabase)

1. 注册 Supabase → 创建项目
2. Settings → Database → Connection string → URI
3. 复制连接字符串，类似：
   ```
   postgresql+asyncpg://postgres.xxx:password@aws-0-xx.pooler.supabase.com:6543/postgres
   ```
4. 修改 `backend/.env`：
   ```
   DATABASE_URL=postgresql+asyncpg://postgres.xxx:password@...supabase.com:6543/postgres
   ```

### 9.3 后端 (Render)

1. Push 代码到 GitHub
2. Render → New Web Service → 连接仓库
3. 配置：
   - Build: `cd backend && pip install -r requirements.txt`
   - Start: `cd backend && uvicorn app.main:app --host 0.0.0.0 --port $PORT`
4. 添加环境变量

### 9.4 前端 (Vercel)

1. Vercel → Import Project → 连接仓库
2. 配置：
   - Framework: Vite
   - Root: `frontend`
3. 添加环境变量 `VITE_API_URL` = Render 后端地址

## 10. 常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 注册报"请求失败" | Vite 代理 IPv6 问题 | vite.config.js 中 target 用 `127.0.0.1` |
| 后端启动报错 | .env 有未知字段 | 删除 .env 中多余的变量 |
| 401 未授权 | Token 过期 | 重新登录 |
| CORS 错误 | 前端域名不在白名单 | config.py 添加 CORS_ORIGINS |
| 数据库锁定 | SQLite 并发写入 | 生产环境切 PostgreSQL |
