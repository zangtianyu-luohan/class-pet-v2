## 1. 项目概述

### 1.1 背景
"学生积分管理系统"是一个面向教师的班级积分管理系统。学生通过课堂表现获得积分，系统提供排行榜、勋章、课堂工具等激励机制，增加课堂趣味性。

### 1.2 功能清单

| 模块 | 功能 |
|------|------|
| 用户认证 | 登录、修改密码、JWT 鉴权、图形验证码、登录锁定 |
| 班级管理 | 创建/编辑/删除班级、班级切换 |
| 学生管理 | 添加/编辑/删除学生、批量导入（文本/Excel）、搜索排序、导出 Excel |
| 积分系统 | 加分/减分、批量操作、积分记录、积分日志查询 |
| 排行榜 | 积分榜、本周榜 |
| 勋章系统 | 创建勋章、颁发勋章、颁发记录 |
| 课堂工具 | 随机点名、计时器、随机分组、积分抽奖、积分兑换、积分提醒、骰子、噪音检测、秒表、红绿灯、随机数 |
| 管理后台 | 数据概览、用户管理、数据库管理、登录日志、积分日志、数据备份 |

## 4. 后端开发详解

### 4.1 核心设计模式

**依赖注入（Dependency Injection）**：FastAPI 的核心设计，通过 `Depends()` 注入数据库会话和当前用户。

```python
# 获取当前用户的依赖
async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(security),
    db: AsyncSession = Depends(get_db),
) -> User:
    # 验证 JWT → 查询用户 → 检查账号有效期 → 返回
```

**异步全链路**：从路由到数据库全链路异步（`async/await`），高并发下不阻塞。

```python
@router.get("/", response_model=list[StudentOut])
async def list_students(
    class_id: int = Query(...),
    user: User = Depends(get_current_user),     # 依赖注入：鉴权
    db: AsyncSession = Depends(get_db),          # 依赖注入：数据库会话
):
    result = await db.execute(select(Student).where(...))
    return result.scalars().all()
```

### 4.2 认证流程

```
登录 → 图形验证码校验 → 生成 JWT Token（有效期 7 天）
    ↓
前端存储 Token → localStorage
    ↓
每次请求 → Axios 拦截器自动添加 Authorization: Bearer <token>
    ↓
后端 → Depends(get_current_user) → 解码 JWT → 查询用户 → 检查账号有效期
```

### 4.3 安全机制

**密码安全**：bcrypt 哈希存储，密码强度校验（≥8位，含大小写字母和数字）。

**登录安全**：图形验证码（数学题，Python PIL 生成）、登录失败锁定（5次失败锁15分钟）、登录日志记录。

**输入安全**：XSS 过滤（学生/积分 Schema 的 field_validator）、SQL 注入防护（SQLAlchemy 参数化查询）、积分范围限制（-1000~1000）。

**响应安全**：CSP 内容安全策略、X-Content-Type-Options、X-Frame-Options、X-XSS-Protection、Referrer-Policy。全局异常处理器隐藏数据库错误细节。

**账号有效期控制**：每个 API 请求都在 `get_current_user` 依赖中检查账号 `expires_at` 字段。过期账号返回 403，前端拦截器自动清除 token 并跳转登录页。

### 4.4 管理后台

管理后台完全独立于前端教师端，通过 `/admin-panel` 访问。使用自托管的 Vue 3 + axios，深色主题。

功能模块：
- 数据概览（系统统计、活动趋势）
- 用户管理（创建/删除/重置密码/设置有效期）
- 数据库管理（查看/编辑/删除所有表数据）
- 登录日志（安全审计）
- 积分日志（全局查询）
- 数据备份（JSON 导出）

```javascript
// class.js — 班级状态

// fetchClasses() → 从 API 获取
// setCurrentClass(id) → 切换并持久化到 localStorage
```

### 5.2 路由守卫

```javascript
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  if (to.meta.requiresAuth && !authStore.isLoggedIn) next('/login')
  else if (to.meta.guest && authStore.isLoggedIn) next('/dashboard')
  else next()
})
```

### 5.3 Axios 拦截器

```javascript
// 请求拦截：自动加 token
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 响应拦截：统一错误处理
api.interceptors.response.use(
  res => res,
  err => {
    const status = err.response?.status
    const detail = err.response?.data?.detail || '请求失败'
    // 登录接口错误不拦截
    if (err.config?.url?.includes('/api/auth/login')) {
      ElMessage.error(detail)
      return Promise.reject(err)
    }
    // 401/403 → 清除 token，跳转登录
    if (status === 401 || status === 403) {
      localStorage.clear()
      window.location.href = '/login'
      return Promise.reject(err)
    }
    ElMessage.error(detail)
    return Promise.reject(err)
  }
)
```

## 10. 常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 登录报验证码错误 | 验证码是数学计算题 | 如「41 - 9 = ?」填写 32 |
| 连续输错密码被锁 | 登录失败锁定机制 | 等待 15 分钟自动解锁 |
| 401 未授权 | Token 过期 | 重新登录 |
| 403 禁止访问 | 账号有效期已到期 | 联系管理员续期 |
| CORS 错误 | 前端域名不在白名单 | config.py 添加 CORS_ORIGINS |
| 数据库锁定 | SQLite 并发写入 | 避免多进程同时写入 |
| bcrypt 版本冲突 | 依赖版本不兼容 | requirements.txt 固定 bcrypt==4.0.1 |
